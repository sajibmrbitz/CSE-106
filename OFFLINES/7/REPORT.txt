Time Complexity Analysis:

1. Base Case Initialization: 
   Initializing the first row and first column of the DP table takes O(N) and O(M) time.
   So total time for this step is O(N+M).

2. DP Table (Recurrence Relation): 
   I used two nested loops to fill the dp[i][j] table.
   dp[i][j]=the minimum cost to convert the first i characters of S into the first j characters of T

   The recurrence relation considers three possible operations and takes the minimum cost:
   dp[i][j]=min(
       dp[i-1][j-1]+cost,   // match (if cost==0) or replace (if cost==R)
       dp[i-1][j]+D,        // delete the character S[i-1]
       dp[i][j-1]+I         // insert the character T[j-1]
   )

   The outer loop runs N times and the inner loop runs M times.
   Inside loops, calculating the minimum of Insertion, Deletion and Substitution costs takes O(1) time.
   So, this step takes O(N*M) time.

3. Backtracking: 
   Starting from the end of the DP table to reconstruct the operations.
   In the worst case, we move cell by cell to the top-left, which looks like O(N+M) time.

   The sequence of operations to matchoutput formats: Insert -> Delete -> Match -> Replace

   Overall Time Complexity = Initialization Time + DP Filling Time + Backtracking Time
                           = O(N+M) + O(N*M) + O(N+M)
                           = O(N*M)

Space Complexity Analysis:
O(N*M) for the 2D vector (DP table), O(N+M) for the operations vector.
Since sequence lengths are up to 5000, overall space complexity can be considered just O(N*M).
